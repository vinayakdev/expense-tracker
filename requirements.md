## Problem Statement

👨‍💻 You are developing a web application to help users track their daily expenses. The application allows users to categorize expenses into different types, such as food, transportation, entertainment, and more. Users can add expenses and view reports based on various criteria. 📊💰💳

### Requirements :

- Create a new project and configure the necessary databases. 🛠️
- Design and implement a database schema that supports the requirements listed below. 📊
- Implement the following features:
    - **User Management:**
        - Allow users to register, log in, and log out of the application. 🙋‍♂️🔑🚪
        - Implement proper validation and error handling for user authentication. 🛡️
    - **Expense Management:**
        - Allow users to add expenses with the following information:
            - Amount (Float)
            - Description (Text)
            - Category (Predefined list of categories)
            - Date (Timestamp)
        - Implement proper validation and error handling for expense creation.
        - Users should be able to view, edit, and delete their expenses.
    - **Reporting:**
        - Implement a feature that allows users to view reports based on the following criteria:
            - Total expenses per category for a specific month. 📈
            - Average daily expenses for a specific month. 📊
            - Total expenses per category for a specific user. 📊
